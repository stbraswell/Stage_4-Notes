Stage 2: Python

Programming

Programs tell computers what to do. These programs are written in programming languages (like Python), which are special languages that are clear and precise. "Natural" languages (like English) would not be great programming languages due to their ambiguity and verbosity.

Python

Python is a programming language which is run in an interpreter. The interpreter runs the program, interprets them, then executes the program that was written in python by running the program in a language that the computer can understand.

Python Grammar

Grammar is very important since computers are stupid! While humans can interpret things like "me hungry", computers can't. So the language that is input as a command must follow specific rules, else what you expected to happen.....won't!

Python Expressions

Python uses "expressions" as statements.
  • use "#" to make a comment in python
  • "print 1" will give "1"
  • "print 2 * 10" will give "20"
  • in python 5 / 2 = 2, but 5.0 / 2 = 2.5 or 5 / 2.0 = 2.5
Keep in mind that things like "print 2 *" will throw an error!

Variables

  • To "assign a variable" is to give a variable a value
  • Variables are assigned this way: "variable = expression"
  • EX: "Troy = 33", where "Troy" is the variable

Strings

  • Strings are a sequence of characters
  • EX: "Hello"
  • You can concatenate strings and Variables: if T = "Troy", Print 'Hello' + T = Hello Troy
  • Print '!' * 3 = !!!

Indexing Strings

  • When indexing strings, you count from left to right, starting with 0
  • SO: 'udacity'[0] --> 'u' | 'udacity'[1+1] --> 'a'
  • you can also use a variable that is a string: so if, name = 'Troy', name[1] -->'r'
  • if your index is greater than the number of letters in your string, you get an error: 'troy'[4] -->error
  • you can use negative numbers too, this will start counting from the end, so if, name = 'troy', name[-2] -->'o'

Selecing Sub Sequences

  • <string>[<expression 1>:<expression 2>], where <expression 1> is the start and <expression 2> is the end
  • EX:word = 'assume'
  • word[4:6] --> me
  • word[3:] --> ume
  • word[:2] --> as
  • word[:] --> assume

Finding Strings in Strings

  • <string1>.find(<string2>), where you are searching <string1> for <string2>
  • Find (aka method) is a built in procedure prvided by python, that operates on stings
  • if the target string is not found you get -1
  • "Test".find("te")--> -1 , due to capitalization
  • "Test".find("Te")--> 0 , because "Te" starts at position 0
  • you can also use variables instead of typing out <string1> or <string2>
  • every string begins with an "empty string" | empty string = ''
  • <string1>.find(<string2>,<number>), this tells find @ what position to start searching, which is useful for finding <string2> after the first time (or second...) it appears in <string1>

Procedures/Functions: Input -> Function -> Output

  • Procedures or functions are a predefined list of steps that take an input, manipulate the input at each step and then produce an output
  • The benefits of using procedures/functions are:
    • Reduction of code duplication
    • Decomposing complex problems into simpler pieces
    • Improving code clarity
    • Reuse code
  • Procedures/functions must be defined using "def"
  • Procedures/functions expect to recieve the same number of inputs they are defined for
  • Procedures/functions need to have the return statement in order to produce an output
Example #1 - Simple procedure to show structure and basic function
def add_two_numbers(number_1,Number_2):
     return number_1 + number_2

print add_two_numbers (4,3) => 7
print add_two_numbers ('hi','troy') => hitroy

Example #2 - Using Prcedures as inputs

x = 2

print square(square(x)) => 16

Comparitors:

  • >= -> greater than or equal to

  • == -> equal to (NOTE: one equal sign is for assignment of varialbes, two equal signs checks for equality

Structured Data: Lists

A list is a sequence of Anything - characters, strings, numbers...even other lists!
  • Lists take the form: <list> = [<expression>,<expression>,...]
  • Empty list => []
  • Elements of a list start from 0 so: [0,1,2,3...]
  • When defining a list, especially a long list, you can split the <expression>'s up
  • You can even index within an indexed list. i.e. list1 = [['Troy','is'],['cool','!']- print list1[1][1] => will print the 2nd element of the 2nd list in the list, in this case '!'
  • Examples of Structured Data: Lists

List Mutations and Aliasing

Lists support Mutations and Aliasing. With mutation you are able to change the value of a list after it has been created. With Aliasing, you can assign a list to two separate names; however if you mutate the list for one, you mutate it for both.
  • Examples of List Mutations and Aliasing

List Operations

List operations allow you to perform certain tasks on lists
  • The "append" operation lets you insert another element into an existing list: <list>.append(<element>)
  • The "plus" operation is like concatination for lists: [1,2] + [3,4] => [1,2,3,4] Note: The "len" operation outputs the number of elements in a list (this also works on strings): len([0,1]) => 2
