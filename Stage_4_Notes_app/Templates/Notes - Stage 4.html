Stage 0: The basics of the Internet

The Internet

The Internet is a collection of computers that talk to each other. The web is a collection of HTML docs. When a user clicks on a link, that link is essentially an address to an HTML file on a server. The browser translates that HTML file into a webpage that the user sees. HTTP is the main protocol of the web. HTML stands for Hypetext Markup Language.

Tags

Tags are instructions to the computer and are not seen outside of the code (e.g. on the webpage). Elements are the complete tag, so from opening to closing. These are also the "markup". Some tags need to be closed, if these types of tags are not closed, then the tag will apply to all text following the opening of the tag.Other tags do not need to be closed, these are called "Void Tags". Tags can also have attributes. These attributes give more dimension to a tag, such as linking a webpage to text.

Whitespace in HTML

Whitespace in HTML doesn't translate 1:1 on the webpage. Spaces seem to be 1:1, but multiple spaces or carrige returns do not. For example, this is what 10 spaces looks like " ".

Inline and Block

There are inline and block elements. Block elements seem to protect their contents by fencing them, like the paragraph element. Block elements are displayed with a line break while inline elements are not. All elements are rendered as boxes, and only some block elements can be container elements for other block elements. For instance, a <div> can have a <p>, an <ol>, a <ul>, another <div>, but an <ol> or <ul> cannot have a <p> or a <br>.

Stage 1: Creating a Structured Document with HTML

Languages of a Webpage

Web pages can can be made up of HTML, CSS and Javascript, which are each a language. A good way to think of these is to think of a house:
  • HTML is like the walls of a house. it is essentially the "structure" or building material.
  • CSS is like the decor - what the house looks like. Think paint on the walls, art, colors used, size and proportion of the furniture.
  • Javascript is like the interactive elements of the house. Think - light switches, doors..

HTML's "tree-like" Structure

This is exactly what it sounds like. You can essentially draw out a web page as a tree; meaning a "div" can branch into a "header" and a "paragraph". Each of those branches can branch even further depending on what is nested inside of them. Elements define what the branches are and these are nested in other elements.

Speaking of Structure

If you look at the HTML code of a web page the structure looks like a wave or zigzags. This kind of structure helps when reading the code to see where sections start and end. While some editors seem to gently create this structure while you are writing the code, it is not a given.

CSS - Adding Style to the Structure

CSS allows you to give your webpage style. This is a separate file that the browser will read and link to the HTML file. Throughout the code you have many different "blocks", you can assign these blocks classes and use the CSS file to give that class a unique style. This is helpful because this allows you to set a style easily for different blocks, without having to repeat the style setup each time.

There is definately a LOT more to learn about CSS. It is clear from the last segment that CSS is a powerful tool and will take time to learn (most likely from playing and failing)

Things to note about CSS
  • CSS = Cascading Stylesheets
  • You can also define your styles in the html file itself by adding <style></style> in the head section.
  • If the section in CSS starts with a "." then it is defining a Class.
  • If the selector is a tag without the brackets (i.e. "em" instead of <em>), then it is defining the style of that tag.
  • Things that can be inherited: color, font, letter spacing, line height, list style, text alignment, text indentation, text transform, visibility, white space and word spacing (probably others?)
  • Things that are not inherited: background, border, display, float and clear, height and width, margin, min/max height & width, outline, overflow, padding, position, text decoration, vertical alignment and z index (probably others?)
  • "The Box Model" - Margin -> Border -> Padding -> Content

Older browsers don't always like CSS, so add this to the beginning of the CSS file:

* {
-webkit-box-sizing: border-box;
-moz-box-sizing: border-box;
-ms-box-sizing: border-box;
box-sizing: border-box;
}

Stage 2: Python

Programming

Programs tell computers what to do. These programs are written in programming languages (like Python), which are special languages that are clear and precise. "Natural" languages (like English) would not be great programming languages due to their ambiguity and verbosity.

Python

Python is a programming language which is run in an interpreter. The interpreter runs the program, interprets them, then executes the program that was written in python by running the program in a language that the computer can understand.

Python Grammar

Grammar is very important since computers are stupid! While humans can interpret things like "me hungry", computers can't. So the language that is input as a command must follow specific rules, else what you expected to happen.....won't!

Python Expressions

Python uses "expressions" as statements.
  • use "#" to make a comment in python
  • "print 1" will give "1"
  • "print 2 * 10" will give "20"
  • in python 5 / 2 = 2, but 5.0 / 2 = 2.5 or 5 / 2.0 = 2.5
Keep in mind that things like "print 2 *" will throw an error!

Variables

  • To "assign a variable" is to give a variable a value
  • Variables are assigned this way: "variable = expression"
  • EX: "Troy = 33", where "Troy" is the variable

Strings

  • Strings are a sequence of characters
  • EX: "Hello"
  • You can concatenate strings and Variables: if T = "Troy", Print 'Hello' + T = Hello Troy
  • Print '!' * 3 = !!!

Indexing Strings

  • When indexing strings, you count from left to right, starting with 0
  • SO: 'udacity'[0] --> 'u' | 'udacity'[1+1] --> 'a'
  • you can also use a variable that is a string: so if, name = 'Troy', name[1] -->'r'
  • if your index is greater than the number of letters in your string, you get an error: 'troy'[4] -->error
  • you can use negative numbers too, this will start counting from the end, so if, name = 'troy', name[-2] -->'o'

Selecing Sub Sequences

  • <string>[<expression 1>:<expression 2>], where <expression 1> is the start and <expression 2> is the end
  • EX:word = 'assume'
  • word[4:6] --> me
  • word[3:] --> ume
  • word[:2] --> as
  • word[:] --> assume

Finding Strings in Strings

  • <string1>.find(<string2>), where you are searching <string1> for <string2>
  • Find (aka method) is a built in procedure prvided by python, that operates on stings
  • if the target string is not found you get -1
  • "Test".find("te")--> -1 , due to capitalization
  • "Test".find("Te")--> 0 , because "Te" starts at position 0
  • you can also use variables instead of typing out <string1> or <string2>
  • every string begins with an "empty string" | empty string = ''
  • <string1>.find(<string2>,<number>), this tells find @ what position to start searching, which is useful for finding <string2> after the first time (or second...) it appears in <string1>

Procedures/Functions: Input -> Function -> Output

  • Procedures or functions are a predefined list of steps that take an input, manipulate the input at each step and then produce an output
  • The benefits of using procedures/functions are:
    • Reduction of code duplication
    • Decomposing complex problems into simpler pieces
    • Improving code clarity
    • Reuse code
  • Procedures/functions must be defined using "def"
  • Procedures/functions expect to recieve the same number of inputs they are defined for
  • Procedures/functions need to have the return statement in order to produce an output

Stage 3: Programming with objects

Object-oriented programming (OOP) is a programming paradigm based on the concept of "objects", which are data structures that contain data, in the form of fields, often known as attributes; and code, in the form of procedures, often known as methods. A distinguishing feature of objects is that an object's procedures can access and often modify the data fields of the object with which they are associated (objects have a notion of "this" or "self"). In OO programming, computer programs are designed by making them out of objects that interact with one another. There is significant diversity in object-oriented programming, but most popular languages are class-based, meaning that objects are instances of classes, which typically also determines their type.

Structured Data: Definitions

  • Class - A neatly packaged box that puts things together really well. A class is like a blueprint of a building - it can be used to build multiple buildings. A user-defined prototype for an object that defines a set of attributes that characterize any object of the class. The attributes are data members (class variables and instance variables) and methods, accessed via dot notation.
  • Instance/Object - A unique instance of a data structure that's defined by its class. An object comprises both data members (class variables and instance variables) and methods. An individual object of a certain class. An object obj that belongs to a class Circle, for example, is an instance of the class Circle.
  • Module - A module allows you to logically organize your Python code. Grouping related code into a module makes the code easier to understand and use. A module is a Python object with arbitrarily named attributes that you can bind and reference. A module is a file consisting of Python code. A module can define functions, classes and variables. A module can also include runnable code.
  • Library - A library is a collection of implementations of behavior, written in terms of a language, that has a well-defined interface by which the behavior is invoked. This means that as long as a higher level program uses a library to make system calls, it does not need to be re-written to implement those system calls over and over again. In addition, the behavior is provided for reuse by multiple independent programs. A program invokes the library-provided behavior via a mechanism of the language.
  • Constructor - When an instance is created, the classes constructor is called (__init__)
  • Self - Used in the constructor, self is the instance and is always the first argument
  • Instance Variable - A variable associated with a specific instance
  • Instance Method - A function defined in a class and associated with an instance.
