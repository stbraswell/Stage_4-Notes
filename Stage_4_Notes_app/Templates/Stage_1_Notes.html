Stage 1: Creating a Structured Document with HTML

Languages of a Webpage

Web pages can can be made up of HTML, CSS and Javascript, which are each a language. A good way to think of these is to think of a house:
  • HTML is like the walls of a house. it is essentially the "structure" or building material.
  • CSS is like the decor - what the house looks like. Think paint on the walls, art, colors used, size and proportion of the furniture.
  • Javascript is like the interactive elements of the house. Think - light switches, doors..

HTML's "tree-like" Structure

This is exactly what it sounds like. You can essentially draw out a web page as a tree; meaning a "div" can branch into a "header" and a "paragraph". Each of those branches can branch even further depending on what is nested inside of them. Elements define what the branches are and these are nested in other elements.

Speaking of Structure

If you look at the HTML code of a web page the structure looks like a wave or zigzags. This kind of structure helps when reading the code to see where sections start and end. While some editors seem to gently create this structure while you are writing the code, it is not a given.

CSS - Adding Style to the Structure

CSS allows you to give your webpage style. This is a separate file that the browser will read and link to the HTML file. Throughout the code you have many different "blocks", you can assign these blocks classes and use the CSS file to give that class a unique style. This is helpful because this allows you to set a style easily for different blocks, without having to repeat the style setup each time.

There is definately a LOT more to learn about CSS. It is clear from the last segment that CSS is a powerful tool and will take time to learn (most likely from playing and failing)

Things to note about CSS
  • CSS = Cascading Stylesheets
  • You can also define your styles in the html file itself by adding <style></style> in the head section.
  • If the section in CSS starts with a "." then it is defining a Class.
  • If the selector is a tag without the brackets (i.e. "em" instead of <em>), then it is defining the style of that tag.
  • Things that can be inherited: color, font, letter spacing, line height, list style, text alignment, text indentation, text transform, visibility, white space and word spacing (probably others?)
  • Things that are not inherited: background, border, display, float and clear, height and width, margin, min/max height & width, outline, overflow, padding, position, text decoration, vertical alignment and z index (probably others?)
  • "The Box Model" - Margin -> Border -> Padding -> Content

Older browsers don't always like CSS, so add this to the beginning of the CSS file:

* {
-webkit-box-sizing: border-box;
-moz-box-sizing: border-box;
-ms-box-sizing: border-box;
box-sizing: border-box;
}
